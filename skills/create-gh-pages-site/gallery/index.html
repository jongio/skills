Install the skill

These templates are scaffolded by the create-gh-pages-site GitHub Copilot skill, part of the jongio/skills collection. Install it once, then ask Copilot for any template — each card shows the exact prompt.

Option A — the skills CLI

# Global, for GitHub Copilot:
npx skills add jongio/skills --skill create-gh-pages-site -g --agent github-copilot

# …or into the current project instead (drop -g):
npx skills add jongio/skills --skill create-gh-pages-site

Option B — the Copilot plugin marketplace

copilot plugin marketplace add jongio/skills
copilot plugin install create-gh-pages-site@jongio-skills

In the Copilot app: Settings → Plugins → Install ▾ → Add marketplace, enter jongio/skills, then install create-gh-pages-site. Reload skills with /skills reload or start a new session.

Use a template

  1. With Copilot — invoke /create-gh-pages-site with the prompt shown on each card, e.g. "an Astro site for octocat/blog". Leave out the framework or repo and the skill will interview you before scaffolding.
  2. With the generator — node scripts/new-site.mjs <template> --repo <owner/name>, or from anywhere with --registry jongio/gh-pages-templates.
  3. Push to main, then set Settings → Pages → Source → GitHub Actions.

Want to see each template running? Browse the live gallery with previews at jongio.github.io/gh-pages-templates.

Submit your own template

Anyone can add a template to the registry. A template is a folder under templates/<name>/ with a template.json manifest, a .github/workflows/deploy.yml using the official Pages actions, base-path handling via the generator's sentinels, and a README.md. Fork jongio/gh-pages-templates, run node scripts/validate.mjs, and open a PR — once merged it appears in the gallery with a live preview.
